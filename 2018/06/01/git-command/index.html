    分支管理策略

    根据廖雪峰的博客内容，这里记录一下通用的分支管理状态

    1. master是最稳定的分支，这条分支仅仅作为版本的发布，其他任何时候不能够动
    2. dev分支为开发分支，这条分支从master的起点分出来，并向master提交版本。
    3. 所有的个人开发分支全部由dev的起点分支分离出来，开发完成以后向dev分支提交。
